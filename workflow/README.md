# Workflow

## git branching model

![](git_branches.png)

Our `git` branching model:

 * `master`/`main`: a working version
 * `develop`: merging topic branches
 * topic branches: where work is done,
   for example, Issue `x` is done on 
   a branch called `issue_x`
